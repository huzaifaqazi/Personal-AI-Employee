# Skill: Organize Files

## Purpose
Categorize, rename, and structure files systematically.

## When to Use
- Cleaning up messy folders
- Organizing project files
- Archiving old work
- Creating folder structures

## Process
1. Analyze current file structure
2. Identify categories and patterns
3. Create logical folder hierarchy
4. Rename files with consistent naming
5. Move files to appropriate locations
6. Create index or catalog

## Naming Conventions
- Use descriptive names
- Include dates: YYYY-MM-DD format
- Use underscores not spaces
- Add version numbers: v1, v2, etc.
- Group by prefix: PROJECT_, REPORT_, etc.

## Folder Structure Best Practices
```
Project/
├── 01_Planning/
├── 02_Research/
├── 03_Development/
├── 04_Testing/
├── 05_Delivery/
└── Archive/
```
